# Специфікація мови AsciiAsm

## 2.1 Пам'ять

### 2.1.1 Директива #memory

```
#memory memory_size[, 'init_char_value']
```

Вказує кількість комірок пам'яті (ціле число > 0) та значення ASCII для ініціалізації всієї пам'яті (опціонально).
Якщо відсутня — за замовчуванням розмір: **100** і пам'ять не ініціалізована.
Повинна бути першим рядком програми, якщо присутня.
Доступ до пам'яті поза межами викликає припинення програми з помилкою.

### 2.1.2 Директива #on_overflow

Ця прагма дозволяє змінювати поведінку процесора при арифметичному переповненні.

`#on_overflow flag` (режим за замовчуванням):
При переповненні встановлюється лише прапор OF у регістрі FLAGS. Програма продовжує виконання наступної інструкції. Це дозволяє програмісту вирішити, що робити (наприклад, використати `JO label`).

`#on_overflow halt`
При будь-якій операції, яка викликає OF = 1 (включаючи MOV, READ), інтерпретатор негайно припиняє виконання з помилкою `Runtime Error: Type Overflow`. Це корисно для початківців, щоб вони негайно побачили логічні помилки.

### 2.1.3 Модель пам'яті

Пам'ять — це лінійний масив клітинок ASCII, індексований з 0.
Кожна клітинка зберігає один символ (ASCII код 32–126 — друковані символи лише).
Мінімальна адресна одиниця — одна клітинка.

> **Увага:** пам'ять **за замовчуванням не ініціалізована** — клітинки містять випадкові ASCII символи.
> Програма повинна записати значення перед першим читанням або використати `#data` для явної ініціалізації.

### 2.1.4 Типи даних

AsciiAsm має п'ять типів. Усі числові типи зі знаком.

| Тип | Клітинок | Формат у пам'яті | Діапазон |
|------|----------|------------------|----------|
| `CHAR` | 1 | сирий ASCII символ | коди 32–126 (друковані) |
| `WORD` | 2 | [знак] + 1-2 цифри | -9..99 |
| `DWORD` | 4 | [знак] + 3-4 цифри | -999..9999 |
| `QWORD` | 8 | [знак] + 7-8 цифр | -9999999..99999999 |
| `TEXT` | змінна | ASCII символи до терміната `$` | — |

#### Формат чисел у пам'яті (WORD / DWORD / QWORD)

Перша клітинка — знак `'-'` для негативних; для позитивних — найбільш значущий розряд або провідний нуль.
Решта клітинок — цифри з провідними нулями, доповнені до повної ширини.

Приклади як ASCII рядки:

```
WORD   -4  →  "-4"
WORD    4  →  "04"
WORD    0  →  "00"

DWORD  -42   →  "-042"
DWORD   42   →  "0042"
DWORD  -999  →  "-999"   # мінімальне значення для типу DWORD
DWORD    0   →  "0000"
DWORD   999  →  "0999"
DWORD  9999  →  "9999"   # максимальне значення для типу DWORD

QWORD  -12345  →  "-0012345"
QWORD   12345  →  "00012345"
```

#### Формат CHAR у пам'яті

Одна клітинка, зберігає символ безпосередньо як ASCII символ.
Арифметика для CHAR переміщує позицію символу в таблиці ASCII.

```
CHAR 'A'  →  "A"    (ASCII 65)
CHAR 'z'  →  "z"    (ASCII 122)
CHAR ' '  →  " "    (ASCII 32)
```

Допустимі значення: ASCII 32–126 (лише друковані символи).
Вихід за межи діапазону під час операцій — переповнення типу з затиском до найближчого допустимого значення.

#### Формат TEXT у пам'яті

Послідовність ASCII символів, завершена символом-терміном `$`.
Розмір не фіксований — визначається позицією `$`.

```
"Hello"  →  "Hello$"   (6 клітинок)
""       →  "$"         (1 клітинка)
```

### 2.1.5 Директива #data — Ініціалізація пам'яті

Директиви `#data` розміщуються після `#memory` та перед першою інструкцією.
Вони записують значення за вказаною абсолютною адресою.

```
#data address, TYPE value[, color]
```

Необов'язковий третій параметр — колір CSS у форматі #RRGGBB, іменова назва кольору (напр. `red`) або #RRGGBBAA.
Коли присутній, усі комірки пам'яті, записані цією директивою, виділяються цим кольором у панелі візуалізації **Пам'ять** IDE, що допомагає користувачам зрозуміти різні сегменти даних з першого погляду.

Синтаксис для кожного типу:

```
#data 0,  WORD -4
#data 2,  DWORD 42,       #4488ff   ; ціле число — голубий відтінок
#data 6,  QWORD 9999999
#data 14, CHAR 'A',        cornflowerblue   ; символ — названий колір
#data 15, TEXT "Hello$",   #44bb77   ; сегмент тексту — зелений відтінок
```

Правила:
- Адреса — невід'ємне десяткове ціле число. Адреса не повинна перевищувати межі пам'яті, інакше виникає помилка під час виконання.
- Директиви виконуються послідовно; пізніша перезаписує попередню.
- Адреса + розмір не повинні перевищувати `#memory`.
- Для `TEXT`: символ `$` обов'язковий у літералі.
- Для `CHAR`: літерал у одинарних лапках (`'A'`).
- Колір — точно шість шістнадцяткових цифр попередні `#` (напр. `#ff0000`), іменова назва кольору CSS або шість цифр + альфа-канал (напр. `#ff0000aa`). Регістр не має значення.
  - Якщо кілька директив охоплюють одну клітинку пам'яті, переважає колір останньої директиви.
  - Колір не впливає на виконання; це лише допомога для візуалізації.

## 2.2 Архітектурний огляд мови AsciiAsm

### 2.2.1 Регістри загального призначення

| Регістр | Умовне призначення |
|----------|------------------|
| `AX` | Акумулятор, результати |
| `BX` | Адреси, вказівники |
| `CX` | Лічильники, цикли |
| `DX` | Допоміжний операнд |

> Умовне призначення — лише конвенція. Будь-який регістр може використовуватися для будь-якої мети.

Кожен регістр має **динамічний тип** — **CHAR** або **ціле число** — визначений останньою операцією запису:

- `MOV reg, imm` або `MOV reg, WORD/DWORD/QWORD [addr]` → тип регістра стає **ціле число**.
- `MOV reg, CHAR 'c'` або `MOV reg, CHAR [addr]` → тип регістра стає **CHAR**.
- `MOV reg, reg2` → копіює как значення так і тип.

Змішування типів CHAR та ціле число заборонено у всіх операціях, **крім** `ADD`/`SUB`
(CHAR ± ціле число = CHAR). Спроба виконати забережену операцію з переміщеною між типами
призводить до помилки виконання: `Runtime Error: Type Mismatch`.

### 2.2.2 Службові регістри (недоступні з коду програми)

| Регістр | Призначення |
|----------|-----------|
| `SLP` | Вказівник лінії вихідного коду — індекс поточної інструкції у вихідному коді (лінія) |
| `FLAGS` | Прапори стану: ZF, SF, OF |

### 2.2.3 Прапори (FLAGS)

| Прапор | Назва | Встановлюється коли... |
|--------|-------|----------------------|
| `ZF` | Прапор нуля | Результат операції = 0 |
| `SF` | Прапор знака | Математичний результат від'ємний (не урізаний після переповнення) |
| `OF` | Прапор переповнення | Результат перевищує діапазон типу |

Прапори оновлюються: `READ`, `MOV`, `ADD`, `SUB`, `CMP`. Усі інші інструкції не змінюють прапори.

## 2.3 Синтаксис

### 2.3.1 Загальні правила

- Коментар — від `;` до кінця рядка.
- Мнемоніки, регістри, типи — нечутливі до регістру.
- Ідентифікатори мітки: `[a-zA-Z_][a-zA-Z0-9_]*`
- Числові літерали: десяткові з або без знака (`42`, `-7`).
- Літерали символів: у одинарних лапках (`'A'`, `' '`).

### 2.3.2 Структура програми

```
[#memory N]
[#data address, TYPE value[, color]]
...

_start:
    інструкції
    HALT

[інші мітки та інструкції]
```

Мітка `_start:` обов'язкова. Мітки у коді: `identifier:` на початку рядка.
Мітки використовуються лише у гілкуючих інструкціях (`JMP`, `JE`, ...).

### 2.3.3 Адресація

| Форма | Значення |
|-------|----------|
| `reg` | значення регістра (тип динамічний) |
| `imm` | числова константа (тип — ціле число) |
| `CHAR 'c'` | константа символу (тип — CHAR) |
| `TYPE [imm]` | пам'ять за абсолютною адресою типу TYPE |
| `TYPE [reg]` | пам'ять за адресою у регістрі типу TYPE |

`TYPE` обов'язковий для кожного доступу до пам'яті через `[...]`.
`CHAR 'c'` може використовуватися як операнд у `MOV reg, CHAR 'c'`, `CMP reg, CHAR 'c'` та `MOV CHAR [addr], 'c'`.

## 2.4 Набір інструкцій

### 2.4.1 MOV — Переміщення даних (копіювання даних)

```nasm
MOV dst, src                ; загальний формат команди
MOV reg, imm                ; reg ← число (тип регістра → ціле число)
MOV reg, CHAR 'c'           ; reg ← символ (тип регістра → CHAR)
MOV reg, reg2               ; reg ← reg2 (копіюється значення і тип)
MOV reg, TYPE [addr]        ; reg ← значення з пам'яті (тип регістра → згідно TYPE)
MOV TYPE [addr], reg        ; пам'ять ← reg (тип регістра повинен відповідати TYPE)
MOV TYPE [addr], imm        ; пам'ять ← константа (TYPE: WORD/DWORD/QWORD, не CHAR)
MOV CHAR [addr], 'c'        ; пам'ять CHAR ← літерал символу
```

`[addr]` — `[imm]` або `[reg]`.

**Правила типів:**
- `MOV WORD/DWORD/QWORD [addr], reg` — reg **повинен** містити ціле число, інакше `Runtime Error: Type Mismatch`.
- `MOV CHAR [addr], reg` — reg **повинен** містити CHAR, інакше `Runtime Error: Type Mismatch`.
- `MOV CHAR [addr], imm` — **заборонено** (`Runtime Error: Type Mismatch`). Використовуйте `MOV CHAR [addr], 'c'`.

MOV оновлює прапори, якщо дані не помістилися у dst та цифри були урізані.

### 2.4.2 Арифметика

Результат зберігається в першому операнді. Прапори `ZF`, `SF`, `OF` оновлюються.
При переповненні `SF` базується на математичному результаті, а не на урізаному.

```nasm
ADD dst, src
SUB dst, src
```

Дозволені комбінації:

| `dst` | `src` | Примітка |
|-------|-------|---------|
| `reg(ціле число)` | `reg(ціле число)` | ціле число ± ціле число = ціле число |
| `reg(ціле число)` | `imm` | ціле число ± ціле число = ціле число |
| `reg(CHAR)` | `imm` | CHAR ± ціле число = CHAR (зсув позиції ASCII) |
| `reg(CHAR)` | `reg(ціле число)` | CHAR ± ціле число = CHAR (зсув позиції ASCII) |
| `reg` | `TYPE [addr]` | TYPE: WORD/DWORD/QWORD; reg повинен бути цілим числом |
| `TYPE [addr]` | `reg(ціле число)` | прочитай → обчисли → запиши повернення; TYPE: WORD/DWORD/QWORD/CHAR |
| `TYPE [addr]` | `imm` | прочитай → обчисли → запиши повернення; TYPE: WORD/DWORD/QWORD/CHAR |

**Забережені комбінації** (помилка виконання `Runtime Error: Type Mismatch`):
- CHAR + CHAR, CHAR − CHAR (будь-яка комбінація, де обидва операнди CHAR)
- ціле число + CHAR, ціле число − CHAR (src це CHAR)
- `reg(CHAR)` + `TYPE [addr]` (CHAR регістр з числовою пам'яттю)

**Арифметика CHAR** — єдина дозволена перехресна операція: CHAR ± ціле число.
Це дозволяє зсувати символ по таблиці ASCII (напр. змінення регістру букви).
Результат поза 32–126 — переповнення (OF=1), результат затискується до найближчої межі діапазону (32 або 126).

Для WORD/DWORD/QWORD: переповнення → OF=1, результат записується як-є використовуючи найменш значущі цифри.

TEXT не підтримує арифметику.

### 2.4.3 CMP — Порівняння

Обчислює `first − second`, оновлює `ZF`, `SF`, `OF`. Не зберігає значення.
При переповненні `SF` базується на математичному результаті, а не на урізаному.

Порівняння дозволено лише в межах одного типу: CHAR з CHAR, ціле число з цілим числом.
Спроба порівняти CHAR з цілим числом → `Runtime Error: Type Mismatch`.

```nasm
CMP reg,          reg2          ; обидва регістри повинні відповідати за типом (CHAR↔CHAR або ціле число↔ціле число)
CMP reg,          imm           ; reg повинен бути цілим числом
CMP reg,          CHAR 'c'      ; reg повинен бути CHAR
CMP reg,          TYPE [addr]   ; тип reg повинен відповідати TYPE (CHAR↔CHAR або ціле число↔ціле число)
CMP TYPE [addr],  reg           ; тип reg повинен відповідати TYPE
CMP TYPE [addr],  imm           ; TYPE: WORD/DWORD/QWORD (не CHAR)
CMP CHAR [addr],  'c'           ; CHAR ↔ CHAR
```

TYPE: WORD / DWORD / QWORD / CHAR. TEXT не підтримується.

### 2.4.4 Гілкування

Усі числові типи зі знаком — лише умови гілкування зі знаком.

```nasm
JMP label   ; безумовний стрибок
JO  label   ; OF = 1  стрибок якщо сталося переповнення
JNO label   ; OF = 0  стрибок якщо немає переповнення
JE  label   ; ZF=1          рівно (==)
JNE label   ; ZF=0          не рівно (!=)
JL  label   ; SF=1, ZF=0    менше ніж (<)
JLE label   ; SF=1 або ZF=1  менше ніж або рівно (<=)
JG  label   ; SF=0, ZF=0    більше ніж (>)
JGE label   ; SF=0 або ZF=1  більше ніж або рівно (>=)
```

### 2.4.5 READ та WRITE — Введення та виведення

Повний синтаксис (операнд пам'яті):

```nasm
READ    TYPE [addr]         ; прочитати зі stdin → пам'ять
WRITE   TYPE [addr]         ; вивести з пам'яті → stdout
WRITELN [TYPE [addr]]       ; вивести з пам'яті → stdout та вивести символ нового рядка наприкінці.
```

WRITELN — може використовуватися без параметрів, у цьому разі просто виводить символ нового рядка.

**Коротка форма WRITE/WRITELN (регістри та константи):**

```nasm
WRITE   reg                 ; вивести значення регістра (ціле число як число, CHAR як символ)
WRITE   imm                 ; вивести числову константу
WRITE   'c'                 ; вивести літерал символу
WRITE   "text"              ; вивести рядковий літерал прямо (末дий '$' вилучено якщо присутній)

WRITELN reg
WRITELN imm
WRITELN 'c'
WRITELN "text"
```

Приклади:

```nasm
MOV  AX, 42
WRITE AX          ; виводить: 42

MOV  BX, CHAR 'Z'
WRITE BX          ; виводить: Z

WRITE 10          ; виводить: 10
WRITE 'C'         ; виводить: C
WRITE "Hello"     ; виводить: Hello
WRITELN "Done"    ; виводить: Done з новим рядком
```

Поведінка за типом (операнд пам'яті):

| Тип | READ | WRITE |
|------|------|-------|
| `WORD` | прочитати число → зберегти у форматі WORD | вивести число без провідних нулів та '+' |
| `DWORD` | прочитати число → зберегти у форматі DWORD | вивести число без провідних нулів та '+' |
| `QWORD` | прочитати число → зберегти у форматі QWORD | вивести число без провідних нулів та '+' |
| `CHAR` | прочитати один символ → зберегти як CHAR | вивести один символ прямо |
| `TEXT` | прочитати рядок → зберегти + `$` | вивести символи до `$` (виключно), без перевірки довжини крім перевірки на виходження за межі #memory |

Додаткові параметри:
```nasm
READ TYPE  [addr],        "prompt"      ; вивести запит перед запитанням вводу
READ TEXT  [addr],        "prompt"      ; введення тексту з запитом, без обмеження довжини
READ TEXT  [addr], imm                  ; прочитати рядок, макс (imm - 1) символів, додати `$` наприкінці
READ TEXT  [addr], imm,  "prompt"      ; введення тексту з максимальною довжиною та запитом
```

Необов'язковий параметр `"prompt"` рядок літерала передається до середовища хоста та відображається користувачу перед запитанням введення (напр. показано у діалогу браузера). Його можна додати до будь-якої форми READ як останній параметр.

Урізування при `READ` коли число не помістилося у тип: зберегти знак, зберегти найменш значущі цифри, які помістилися у даний тип. При урізуванні встановлюється прапор переповнення OF.

### 2.4.6 HALT

`HALT` — припиняє виконання програми і повинен бути явно вказаний для завершення програми.

## 2.5 Приклади програм

### Сума двох чисел

```nasm
#memory 16
#data 0, DWORD 0
#data 4, DWORD 0

_start:
  READ  DWORD [0], "Enter first number:"
  READ  DWORD [4], "Enter second number:"
  MOV   AX, DWORD [0]
  ADD   AX, DWORD [4]
  MOV   DWORD [0], AX
  WRITE "Sum: "
  WRITELN DWORD [0]
  HALT
```

### Максимум з двох чисел

```nasm
#memory 16
#data 0, DWORD 0
#data 4, DWORD 0

_start:
  READ DWORD [0], "Enter first number:"
  READ DWORD [4], "Enter second number:"
  MOV  AX, DWORD [0]
  CMP  AX, DWORD [4]
  JGE  show_first
  WRITE "Maximum: "
  WRITELN DWORD [4]
  JMP  done
show_first:
  WRITE "Maximum: "
  WRITELN DWORD [0]
done:
  HALT
```

### Цикл: сума від 1 до N

```nasm
#memory 32
#data 0, DWORD 0    ; n
#data 4, DWORD 0    ; i
#data 8, QWORD 0    ; total

_start:
  READ  DWORD [0], "Enter upper limit (N):"
  MOV   DWORD [4], 1     ; i = 1
loop:
  MOV   AX, DWORD [4]
  CMP   AX, DWORD [0]
  JG    done
  ADD   QWORD [8], AX    ; total += i
  ADD   DWORD [4], 1     ; i++
  JMP   loop
done:
  WRITE "Sum from 1 to N: "
  WRITELN QWORD [8]
  HALT
```

### Робота з CHAR

```nasm
#memory 8
#data 0, CHAR 'A'

_start:
  WRITE "Source char is: "
  WRITELN CHAR [0]         ; виводить: A
  ADD   CHAR [0], 1      ; 'A' + 1 = 'B'
  WRITE "Next Char after 'A' is: "
  WRITELN CHAR [0]       ; виводить: B + новий рядок
  HALT
```
