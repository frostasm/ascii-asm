# Специфікація мови AsciiAsm

## 2.1 Пам'ять

### 2.1.1 Директива #memory

```
#memory memory_size[, 'init_char_value']
```

Визначає кількість комірок пам'яті (ціле число > 0) і ASCII-значення для ініціалізації всієї пам'яті (необов'язково).
Якщо директива відсутня, розмір за замовчуванням: **100**, а пам'ять не ініціалізується.
Якщо присутня, має бути першим рядком програми.
Доступ до пам'яті за межами її розміру призводить до завершення програми з помилкою.

### 2.1.2 Директива #on\_overflow

Ця прагма дозволяє змінювати поведінку процесора при арифметичному переповненні.

`#on_overflow flag` (режим за замовчуванням):
При переповненні встановлюється лише прапорець OF у регістрі FLAGS. Програма продовжує виконання з наступної інструкції. Це дозволяє програмісту самостійно вирішити, що робити далі (наприклад, використати `JO label`).

`#on_overflow halt`
Для будь-якої операції, що встановлює OF = 1 (включно з MOV, READ), інтерпретатор негайно зупиняє виконання з помилкою `Runtime Error: Type Overflow`. Це корисно для початківців, щоб одразу бачити логічні помилки.

### 2.1.3 Модель пам'яті

Пам'ять є лінійним масивом ASCII-комірок, пронумерованих з 0.
Кожна комірка зберігає один символ (ASCII-код 32-126, лише друковані символи).
Мінімальна адресована одиниця - одна комірка.

> **Попередження:** за замовчуванням пам'ять **не ініціалізована**, тому комірки містять випадкові ASCII-символи.
> Програма має записати значення перед першим читанням або використати `#data` для явної ініціалізації.

### 2.1.4 Типи даних

AsciiAsm має п'ять типів. Усі числові типи є знаковими.

| Тип | Комірок | Формат у пам'яті | Діапазон |
|-----|---------|------------------|----------|
| `CHAR` | 1 | сирий ASCII-символ | коди 32-126 (друковані) |
| `WORD` | 2 | \[знак] + 1-2 цифри | -9..99 |
| `DWORD` | 4 | \[знак] + 3-4 цифри | -999..9999 |
| `QWORD` | 8 | \[знак] + 7-8 цифр | -9999999..99999999 |
| `TEXT` | змінна | ASCII-символи до термінатора `$` | — |

#### Формат чисел у пам'яті (WORD / DWORD / QWORD)

Перша комірка - знак `'-'` для від'ємних чисел; для додатних - найстарша цифра або початковий нуль.
Решта комірок - цифри, доповнені початковими нулями до повної ширини.

Приклади як ASCII-рядки:

```
WORD   -4  →  "-4"
WORD    4  →  "04"
WORD    0  →  "00"

DWORD  -42   →  "-042"
DWORD   42   →  "0042"
DWORD  -999  →  "-999"   # мінімальне значення для типу DWORD
DWORD    0   →  "0000"
DWORD   999  →  "0999"
DWORD  9999  →  "9999"   # максимальне значення для типу DWORD

QWORD  -12345  →  "-0012345"
QWORD   12345  →  "00012345"
```

#### Формат CHAR у пам'яті

Одна комірка, яка зберігає символ безпосередньо як ASCII-символ.
Арифметика над CHAR зсуває позицію символу в таблиці ASCII.

```
CHAR 'A'  →  "A"    (ASCII 65)
CHAR 'z'  →  "z"    (ASCII 122)
CHAR ' '  →  " "    (ASCII 32)
```

Допустимі значення: ASCII 32-126 (лише друковані символи).
Вихід за межі діапазону під час операцій спричиняє переповнення типу з обмеженням до найближчого допустимого значення.

#### Формат TEXT у пам'яті

Послідовність ASCII-символів, завершена символом-термінатором `$`.
Розмір не є фіксованим - він визначається позицією `$`.

```
"Hello"  →  "Hello$"   (6 комірок)
""       →  "$"        (1 комірка)
```

### 2.1.5 Директива #data — Ініціалізація пам'яті

Директиви `#data` розміщуються після `#memory` і перед першою інструкцією.
Вони записують значення за вказаною абсолютною адресою.

```
#data address, TYPE value[, #RRGGBB]
```

Необов'язковий третій параметр `#RRGGBB` - це CSS hex-літерал кольору (шість шістнадцяткових цифр у нижньому або верхньому регістрі
після символу `#`). Якщо його вказано, усі комірки пам'яті, записані цією директивою, підсвічуються цим
кольором тла на панелі візуалізації **Memory** в IDE, що допомагає користувачам швидко візуально
розрізняти різні сегменти даних.

Синтаксис для кожного типу:

```
#data 0,  WORD -4
#data 2,  DWORD 42,       #4488ff   ; ціле число — синій відтінок
#data 6,  QWORD 9999999
#data 14, CHAR 'A',       #ffaa00   ; символ — бурштиновий відтінок
#data 15, TEXT "Hello$",  #44bb77   ; текстовий сегмент — зелений відтінок
```

Правила:

* Адреса - невід'ємне десяткове ціле число. Адреса не повинна виходити за межі пам'яті, інакше виникає помилка виконання.
* Директиви виконуються послідовно; пізніша директива перезаписує попередню.
* Адреса + розмір не повинні перевищувати `#memory`.
* Для `TEXT`: символ `$` є обов'язковим у літералі.
* Для `CHAR`: літерал записується в одинарних лапках (`'A'`).
* Колір - рівно шість шістнадцяткових цифр після `#` (наприклад, `#ff0000`). Регістр не має значення.
  * Якщо кілька директив покривають ту саму комірку пам'яті, використовується колір останньої директиви.
  * Колір не впливає на виконання програми; це лише засіб візуалізації.

## 2.2 Архітектурний огляд мови AsciiAsm

### 2.2.1 Регістри загального призначення

| Регістр | Традиційне призначення |
|---------|------------------------|
| `AX` | Акумулятор, результати |
| `BX` | Адреси, вказівники |
| `CX` | Лічильники, цикли |
| `DX` | Допоміжний операнд |
| `SI` | Індекс джерела, вказівники на джерело |
| `DI` | Індекс призначення, вказівники на призначення |

> Традиційне призначення - лише умовність. Будь-який регістр можна використовувати для будь-якої мети.

Кожен регістр має **динамічний тип** - **CHAR** або **ціле число** - який визначається
останньою операцією запису:

* `MOV reg, imm` або `MOV reg, WORD/DWORD/QWORD [addr]` → тип регістра стає **цілим числом**.
* `MOV reg, label` → тип регістра стає **цілим числом** і зберігає вказівник інструкції на першу інструкцію після цієї мітки.
* `MOV reg, CHAR 'c'` або `MOV reg, CHAR [addr]` → тип регістра стає **CHAR**.
* `MOV reg, reg2` → копіює і значення, і тип.

Змішування типів CHAR і цілих чисел заборонене в усіх операціях, **крім** `ADD`/`SUB`
(CHAR ± ціле число = CHAR). Спроба виконати заборонену міжтипову операцію
призводить до помилки виконання: `Runtime Error: Type Mismatch`.

### 2.2.2 Службові регістри (недоступні з коду програми)

| Регістр | Призначення |
|---------|-------------|
| `SLP` | Source Line Pointer - індекс поточної інструкції у вихідному коді (рядок) |
| `FLAGS` | Прапорці стану: ZF, SF, OF |

### 2.2.3 Прапорці (FLAGS)

| Прапорець | Назва | Встановлюється коли... |
|-----------|-------|------------------------|
| `ZF` | Zero Flag | Результат операції = 0 |
| `SF` | Sign Flag | Математичний результат є від'ємним (без усічення після переповнення) |
| `OF` | Overflow Flag | Результат виходить за межі діапазону типу |

Прапорці оновлюються інструкціями: `READ`, `MOV`, `ADD`, `SUB`, `CMP`. Усі інші інструкції прапорці не змінюють.

## 2.3 Синтаксис

### 2.3.1 Загальні правила

* Коментар - від `;` до кінця рядка.
* Мнемоніки, регістри, типи - нечутливі до регістру.
* Ідентифікатори міток: `[a-zA-Z_][a-zA-Z0-9_]*`
* Числові літерали: десяткові зі знаком або без (`42`, `-7`).
* Символьні літерали: в одинарних лапках (`'A'`, `' '`).

### 2.3.2 Структура програми

```
[#memory N]
[#data address, TYPE value[, #RRGGBB]]
...

_start:
    instructions
    HALT

[other labels and instructions]
```

Мітка `_start:` є обов'язковою. Мітки в коді: `identifier:` на початку рядка.
Коли мітка використовується як операнд, вона перетворюється на вказівник інструкції першої інструкції після цієї мітки.
Мітки можна використовувати безпосередньо в інструкціях переходу (`JMP`, `JE`, ...), а також завантажувати в цілий регістр через `MOV reg, label`.

### 2.3.3 Адресація

| Форма | Значення |
|-------|----------|
| `reg` | значення регістра (тип динамічний) |
| `imm` | числова константа (тип - ціле число) |
| `label` | вказівник інструкції першої інструкції після мітки (тип - ціле число) |
| `CHAR 'c'` | символьна константа (тип - CHAR) |
| `TYPE [imm]` | доступ до пам'яті типу "TYPE" за числовою константною адресою imm |
| `TYPE [reg]` | доступ до пам'яті типу "TYPE" за адресою, взятою з регістра reg |

`TYPE` є обов'язковим для кожного доступу до пам'яті через `[...]`.
`CHAR 'c'` можна використовувати як операнд у `MOV reg, CHAR 'c'`, `CMP reg, CHAR 'c'` та `MOV CHAR [addr], 'c'`.

## 2.4 Набір інструкцій

### 2.4.1 MOV — Переміщення даних (копіювання даних)

```nasm
MOV dst, src                ; загальний формат команди
MOV reg, imm                ; reg ← число (тип регістра → ціле число)
MOV reg, label              ; reg ← вказівник інструкції мітки (тип регістра → ціле число)
MOV reg, CHAR 'c'           ; reg ← символ (тип регістра → CHAR)
MOV reg, reg2               ; reg ← reg2 (копіюються значення і тип)
MOV reg, TYPE [addr]        ; reg ← значення з пам'яті (тип регістра → відповідно до TYPE)
MOV TYPE [addr], reg        ; пам'ять ← reg (тип регістра має відповідати TYPE)
MOV TYPE [addr], imm        ; пам'ять ← константа (TYPE: WORD/DWORD/QWORD, не CHAR)
MOV CHAR [addr], 'c'        ; CHAR у пам'яті ← символьний літерал
```

`[addr]` - це `[imm]` або `[reg]`.

**Правила типів:**

* `MOV reg, label` - reg отримує цілий вказівник інструкції на першу інструкцію після мітки.
* `MOV WORD/DWORD/QWORD [addr], reg` - reg **повинен** містити ціле число, інакше `Runtime Error: Type Mismatch`.
* `MOV CHAR [addr], reg` - reg **повинен** містити CHAR, інакше `Runtime Error: Type Mismatch`.
* `MOV CHAR [addr], imm` - **заборонено** (`Runtime Error: Type Mismatch`). Використовуйте `MOV CHAR [addr], 'c'`.

MOV оновлює прапорці, якщо дані не поміщаються в dst і цифри було усічено.

### 2.4.2 Арифметика

Результат зберігається в першому операнді. Прапорці `ZF`, `SF`, `OF` оновлюються.
При переповненні `SF` базується на математичному результаті, а не на усіченому.

```nasm
ADD dst, src
SUB dst, src
```

Дозволені комбінації:

| `dst` | `src` | Примітка |
|-------|-------|----------|
| `reg(integer)` | `reg(integer)` | ціле ± ціле = ціле |
| `reg(integer)` | `imm` | ціле ± ціле = ціле |
| `reg(CHAR)` | `imm` | CHAR ± ціле = CHAR (зсув позиції в ASCII) |
| `reg(CHAR)` | `reg(integer)` | CHAR ± ціле = CHAR (зсув позиції в ASCII) |
| `reg` | `TYPE [addr]` | TYPE: WORD/DWORD/QWORD; reg має бути цілим числом |
| `TYPE [addr]` | `reg(integer)` | читання → обчислення → зворотний запис; TYPE: WORD/DWORD/QWORD/CHAR |
| `TYPE [addr]` | `imm` | читання → обчислення → зворотний запис; TYPE: WORD/DWORD/QWORD/CHAR |

**Заборонені комбінації** (помилка виконання `Runtime Error: Type Mismatch`):

* CHAR + CHAR, CHAR − CHAR (будь-яка комбінація, де обидва операнди є CHAR)
* ціле + CHAR, ціле − CHAR (`src` має тип CHAR)
* `reg(CHAR)` + `TYPE [addr]` (CHAR-регістр із числовою пам'яттю)

**Арифметика CHAR** - єдиний дозволений міжтиповий випадок: CHAR ± ціле число.
Це дозволяє зсувати символ уздовж таблиці ASCII (наприклад, змінювати регістр літери).
Результат поза межами 32-126 означає переповнення (OF=1), а значення обмежується найближчою межею діапазону (32 або 126).

Для WORD/DWORD/QWORD: переповнення → OF=1, результат записується як є, використовуючи молодші значущі цифри.

TEXT не підтримує арифметику.

### 2.4.3 CMP — Порівняння

Обчислює `first − second`, оновлює `ZF`, `SF`, `OF`. Значення не зберігається.
При переповненні `SF` базується на математичному результаті, а не на усіченому.

Порівняння дозволене лише в межах одного типу: CHAR з CHAR, ціле число з цілим числом.
Спроба порівняти CHAR з цілим числом → `Runtime Error: Type Mismatch`.

```nasm
CMP reg,          reg2          ; обидва регістри мають збігатися за типом (CHAR↔CHAR або integer↔integer)
CMP reg,          imm           ; reg має бути цілим числом
CMP reg,          CHAR 'c'      ; reg має бути CHAR
CMP reg,          TYPE [addr]   ; тип reg має відповідати TYPE (CHAR↔CHAR або integer↔integer)
CMP TYPE [addr],  reg           ; тип reg має відповідати TYPE
CMP TYPE [addr],  imm           ; TYPE: WORD/DWORD/QWORD (не CHAR)
CMP CHAR [addr],  'c'           ; CHAR ↔ CHAR
```

TYPE: WORD / DWORD / QWORD / CHAR. TEXT не підтримується.

### 2.4.4 Переходи

Усі числові типи є знаковими, тому умови переходу теж лише знакові.
Ціллю переходу може бути:

* мітка, яка перетворюється на вказівник інструкції першої інструкції після цієї мітки
* цілий регістр, що містить коректний вказівник інструкції

Використання CHAR-регістра як цілі переходу спричиняє `Runtime Error: Type Mismatch`.
Використання цілого значення, яке не вказує на валідну інструкцію, спричиняє помилку виконання.

```nasm
JMP target   ; безумовний перехід
JO  target   ; OF = 1  перейти, якщо відбулося переповнення
JNO target   ; OF = 0  перейти, якщо переповнення не було
JE  target   ; ZF=1          дорівнює (==)
JNE target   ; ZF=0          не дорівнює (!=)
JL  target   ; SF=1, ZF=0    менше (<)
JLE target   ; SF=1 or ZF=1  менше або дорівнює (<=)
JG  target   ; SF=0, ZF=0    більше (>)
JGE target   ; SF=0 or ZF=1  більше або дорівнює (>=)
```

Приклад:

```nasm
  MOV AX, fn_sum    ; AX ← вказівник інструкції на першу інструкцію під fn_sum
  JMP AX            ; непрямий перехід через AX

fn_sum:
  ; some code
```

### 2.4.5 READ та WRITE — Ввід і вивід

Повний синтаксис (операнд пам'яті):

```nasm
READ    TYPE [addr]         ; читання зі stdin → пам'ять
WRITE   TYPE [addr]         ; вивід із пам'яті → stdout
WRITELN [TYPE [addr]]       ; вивід із пам'яті → stdout і символ нового рядка наприкінці.
```

WRITELN можна використовувати без параметрів; у такому разі вона просто виводить символ нового рядка.

**Скорочені форми WRITE/WRITELN (регістри та константи):**

```nasm
WRITE   reg                 ; вивести значення регістра (ціле як число, CHAR як символ)
WRITE   imm                 ; вивести цілу константу
WRITE   'c'                 ; вивести символьний літерал
WRITE   "text"              ; вивести рядковий літерал безпосередньо (кінцевий '$', якщо є, відкидається)

WRITELN reg
WRITELN imm
WRITELN 'c'
WRITELN "text"
```

Приклади:

```nasm
MOV  AX, 42
WRITE AX          ; виведе: 42

MOV  BX, CHAR 'Z'
WRITE BX          ; виведе: Z

WRITE 10          ; виведе: 10
WRITE 'C'         ; виведе: C
WRITE "Hello"     ; виведе: Hello
WRITELN "Done"    ; виведе: Done і потім новий рядок
```

Поведінка за типом (операнд пам'яті):

| Тип | READ | WRITE |
|-----|------|-------|
| `WORD` | зчитати число → зберегти у форматі WORD | вивести число без початкових нулів і `+` |
| `DWORD` | зчитати число → зберегти у форматі DWORD | вивести число без початкових нулів і `+` |
| `QWORD` | зчитати число → зберегти у форматі QWORD | вивести число без початкових нулів і `+` |
| `CHAR` | зчитати один символ → зберегти як CHAR | вивести один символ напряму |
| `TEXT` | зчитати рядок → зберегти + `$` | вивести символи до `$` (не включно), без перевірки довжини, окрім перевірки виходу за межі `#memory` |

Додаткові параметри:

```nasm
READ TYPE  [addr],        "prompt"      ; показати запрошення перед зчитуванням вводу
READ TEXT  [addr],        "prompt"      ; ввід TEXT із запрошенням, без обмеження довжини
READ TEXT  [addr], imm                  ; зчитати рядок, максимум (imm - 1) символів, додати `$` наприкінці
READ TEXT  [addr], imm,  "prompt"       ; ввід TEXT з максимальною довжиною і запрошенням
```

Необов'язковий рядковий літерал `"prompt"` передається хост-середовищу і відображається користувачу перед запитом вводу (наприклад, у діалозі браузера). Його можна додавати до будь-якої форми READ як останній параметр.

Усічення в `READ`, коли число не поміщається в тип: знак зберігається, а значущі молодші цифри, які вміщаються в тип, залишаються. При усіченні встановлюється прапорець переповнення OF.

### 2.4.6 HALT

`HALT` - зупиняє виконання програми і має бути явно вказаний для її завершення.

***

## 2.5 Приклади програм

### Сума двох чисел

```nasm
#memory 16
#data 0, DWORD 0
#data 4, DWORD 0

_start:
  READ  DWORD [0], "Enter first number:"
  READ  DWORD [4], "Enter second number:"
  MOV   AX, DWORD [0]
  ADD   AX, DWORD [4]
  MOV   DWORD [0], AX
  WRITE "Sum: "
  WRITELN DWORD [0]
  HALT
```

### Максимум із двох чисел

```nasm
#memory 16
#data 0, DWORD 0
#data 4, DWORD 0

_start:
  READ DWORD [0], "Enter first number:"
  READ DWORD [4], "Enter second number:"
  MOV  AX, DWORD [0]
  CMP  AX, DWORD [4]
  JGE  show_first
  WRITE "Maximum: "
  WRITELN DWORD [4]
  JMP  done
show_first:
  WRITE "Maximum: "
  WRITELN DWORD [0]
done:
  HALT
```

### Цикл: сума від 1 до N

```nasm
#memory 32
#data 0, DWORD 0    ; n
#data 4, DWORD 0    ; i
#data 8, QWORD 0    ; total

_start:
  READ  DWORD [0], "Enter upper limit (N):"
  MOV   DWORD [4], 1     ; i = 1
loop:
  MOV   AX, DWORD [4]
  CMP   AX, DWORD [0]
  JG    done
  ADD   QWORD [8], AX    ; total += i
  ADD   DWORD [4], 1     ; i++
  JMP   loop
done:
  WRITE "Sum from 1 to N: "
  WRITELN QWORD [8]
  HALT
```

### Виведення рядка

```nasm
#memory 32
#data 0, TEXT "Hello, World!$"

_start:
  WRITELN TEXT [0]
  HALT
```

### Робота з CHAR

```nasm
#memory 8
#data 0, CHAR 'A'

_start:

  WRITE "Source char is: "
  WRITELN CHAR [0]         ; виведе: A
  ADD   CHAR [0], 1        ; 'A' + 1 = 'B'
  WRITE "Next Char after 'A' is: "
  WRITELN CHAR [0]         ; виведе: B + новий рядок
  HALT
```

## 2.6 Підсумкова таблиця інструкцій

| Інструкція | Опис | FLAGS |
|------------|------|:-----:|
| `MOV dst, src` | Переміщення (типи мають збігатися; `MOV reg, label` зберігає вказівник інструкції) | Yes |
| `ADD dst, src` | Додавання (дозволено CHAR±integer) | Yes |
| `SUB dst, src` | Віднімання (дозволено CHAR±integer) | Yes |
| `CMP a, b` | Порівняння (типи мають збігатися) | Yes |
| `JMP target` | Безумовний перехід до мітки або цілі в цілому регістрі | No |
| `JE / JNE target` | Дорівнює / не дорівнює; ціль - мітка або цілий регістр | No |
| `JL / JLE target` | Менше / менше або дорівнює; ціль - мітка або цілий регістр | No |
| `JG / JGE target` | Більше / більше або дорівнює; ціль - мітка або цілий регістр | No |
| `JO / JNO target` | Переповнення / без переповнення; ціль - мітка або цілий регістр | No |
| `READ TYPE [addr]` | Зчитати зі stdin у пам'ять | Yes |
| `READ TYPE [addr], "msg"` | Зчитати зі stdin з текстом запрошення | Yes |
| `READ TEXT [addr], n` | Зчитати рядок (максимум n-1 символів) | Yes |
| `READ TEXT [addr], n, "msg"` | Зчитати рядок з максимальною довжиною і запрошенням | Yes |
| `WRITE TYPE [addr]` | Вивести з пам'яті у stdout | No |
| `WRITE reg` | Вивести значення регістра у stdout | No |
| `WRITE imm` | Вивести цілу константу у stdout | No |
| `WRITE 'c'` | Вивести символьний літерал у stdout | No |
| `WRITE "text"` | Вивести рядковий літерал у stdout | No |
| `WRITELN TYPE [addr]` | Вивести з пам'яті у stdout + "\n" | No |
| `WRITELN reg/imm/'c'/"text"` | Вивести значення + "\n" | No |
| `WRITELN` | Вивести лише новий рядок | No |
| `HALT` | Зупинити програму | No |

## 2.7 Відкриті питання для майбутніх версій

### Змінні

На поточному етапі AsciiAsm не має іменованих змінних. Дані адресуються
числовим способом (`[0]`, `[4]`, `[BX]`), а ініціалізація виконується через `#data`.

У майбутніх версіях плануються іменовані змінні:

```nasm
; Можливий синтаксис у майбутньому:
x   DWORD 0
msg TEXT  "Hello$"
```

Іменована змінна автоматично резервуватиме адресу і дозволятиме
звертатися до неї за ім'ям: `MOV AX, DWORD [x]`.

### Інші заплановані можливості

* **Перетворення типів** - інструкція `CAST` для явного перетворення integer↔CHAR (через ASCII-код).
* **Стек і підпрограми** - `PUSH`, `POP`, `CALL`, `RET`.
* **Множення / ділення** - `MUL`, `DIV`.
* **Адресація зі зміщенням** - `[BX + 4]` для роботи з масивами.
* **Макроси** - визначення власних скорочень.
